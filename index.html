I used the GRCh38.p13 human genome gtf and fasta files from GenCode (PRI), see details below for how to obtain those.

Links for used packages and websites with versions used:

  • FastQC (Quality check). I used version 0.11.9 https://www.bioinformatics.babraham.ac.uk/projects/fastqc/
  • Trim_galore (fastq.gz trimming). I used version 0.6.7. https://github.com/FelixKrueger/TrimGalore/blob/master/Docs/Trim_Galore_User_Guide.md#step-2-adapter-trimming
  • Samtools (index aligned .bam files): I used version 1.15 http://www.htslib.org/
  • bowtie2 (alignment). I used version 2.4.5 http://bowtie-bio.sourceforge.net/bowtie2/index.shtml
  • Deeptools bamCoverage (bw file generation). I used version 3.8.5-1 https://deeptools.readthedocs.io/en/develop/content/tools/bamCoverage.html#Read%20coverage%20normalization%20options
  • macs2 (for peakcalling) : https://pypi.org/project/MACS2/ or also https://manpages.ubuntu.com/manpages/xenial/man1/macs2_callpeak.1.html

A couple other useful tools:

  • GREAT (to see where your peaks are in relation to transcriptional start sites of genes): https://great.stanford.edu/great/public/html/
  • homer (to annotate peaks with nearby genes): I used version 4.11.1 http://homer.ucsd.edu/homer/ngs/annotation.html
  • bedtools (to intersect narrowPeak files to find overlap) https://bedtools.readthedocs.io/en/latest/
  • Conversion ENSEMBL identifiers to gene symbol: https://www.biotools.fr/mouse/ensembl_symbol_converter
  • Venn overlaps: https://bioinformatics.psb.ugent.be/webtools/Venn/
  • GO term analysis for gene lists: http://geneontology.org/#:~:text=The%20Gene%20Ontology%20(GO)%20knowledgebase,genetics%20experiments%20in%20biomedical%20research

This logbook provides instructions for running our ChIP-seq analysis pipeline on a high performance computing cluster such as the Biowulf linux cluster. These commands may need to be modified to run on other machines.

FastQC Analysis

To check the quality of the sequencing data we run a program called FastQC, which give you a graphical output to inspect the data quality in form of a .html file. Read the FastQC documentation see link above or copied here: https://www.bioinformatics.babraham.ac.uk/projects/fastqc/ to understand the output better. As FastQC is relatively quick to run I just use a basic bash script I write in nano and run it like this. The script specifies the location of input directory (dir). Caution/little exercise: It may be the output does not land in the folder we want it but in the input directory; we can fix the script if so (or just move the files manually).

First, create a FastQC.sh bash script.

In[]:
nano FastQC_paired.sh; nano FastQC_single.sh
In[]:
#!/bin/bash
#SBATCH --job-name=FastQCpaired
#SBATCH --ntasks=1
#SBATCH --cpus-per-task=32
#SBATCH --time=0-64:00:00
#SBATCH --output=outfile.%j
#SBATCH --error=errfile.%j
dir=/data/gallegosda/GEO/Published_ChIP-seq/fastq/paired
module load fastqc; cd $dir; mkdir FastQC; cd FastQC; fastqc ../*.fastq.gz

Submit the batch script.

In[]:
sbatch FastQC_paired.sh; sbatch FastQC_single.sh

TrimGalore; NOTE: today I tried TrimGalore on everything

See manual: https://github.com/FelixKrueger/TrimGalore/blob/master/Docs/Trim_Galore_User_Guide.md#step-2-adapter-trimming

USAGE: trim_galore [options] <filename(s)>

  • --paired
    • This option performs length trimming of quality/adapter/RRBS trimmed reads for paired-end files. To pass the validation test, both sequences of a sequence pair are required to have a certain minimum length which is governed by the option --length (see above). If only one read passes this length threshold the other read can be rescued (see option --retain_unpaired).
    • Using this option lets you discard too short read pairs without disturbing the sequence-by-sequence order of FastQ files which is required by many aligners. Trim Galore! expects paired-end files to be supplied in a pairwise fashion, e.g. file1_1.fq file1_2.fq SRR2_1.fq.gz SRR2_2.fq.gz ... .
